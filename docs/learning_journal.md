# 🐣 응애 개발자의 배움 노트

이 파일은 앱을 만들면서 배운 중요한 개념들과 설명을 모아두는 곳입니다. 언제든 다시 열어보며 복습하세요!

---

## 1. 내 컴퓨터에서 앱 실행하기 (Environment Setup)

### 상황
Google AI Studio에서 만든 앱을 내 컴퓨터(로컬 환경)로 가져왔습니다.

### 배움
- **의존성 설치 (`npm install`)**: 앱이 실행되려면 여러 가지 부품(라이브러리)이 필요합니다. 이 명령어는 인터넷에서 필요한 부품들을 다 받아와서 조립할 준비를 하는 과정입니다.
- **앱 실행 (`npm run dev`)**: 개발 모드로 앱을 실행합니다. 이제 내 컴퓨터가 작은 서버가 되어 `localhost:3000` 주소로 접속할 수 있게 됩니다.

## 2. 리팩토링: 원룸에서 아파트로 (Refactoring)

### 상황
`App.tsx` 파일 하나에 모든 코드가 다 들어있어서 파일이 너무 길고 복잡했습니다 (1200줄 이상).

### 비유: "원룸 vs 방 여러 개"
- **이전 (`App.tsx` 몰빵)**: 커다란 원룸에 침대, 부엌, 화장실, 책상이 다 섞여 있는 상태. 뭐 하나 찾으려면 온 방을 다 뒤져야 합니다.
- **이후 (컴포넌트 분리)**:
    - `App.tsx`: **거실** (전체적인 이동 통로)
    - `LandingView.tsx`: **현관** (첫인상)
    - `BrainDumpView.tsx`: **서재** (생각 정리)
    - `DashboardView.tsx`: **안방** (메인 활동 공간)
    - ...등등
    
이제 "현관을 꾸미고 싶으면" `LandingView.tsx`만 찾아가면 됩니다. 코드를 관리하기 훨씬 쉬워졌습니다!

## 3. API 키: 디지털 열쇠 (API Keys)

### 상황
앱을 다운로드받았는데 AI 기능이 작동하지 않았습니다.

### 비유: "집 열쇠 복사"
- 구글 AI 스튜디오에서 코드를 다운로드받는 건 **"모델하우스를 통째로 트럭에 실어 내 땅으로 가져오는 것"**과 같습니다.
- 하지만 보안상 **"집 열쇠(API 키)"**는 같이 딸려오지 않습니다. 열쇠가 없으니 대문이 안 열리는 것이죠.
- **해결책**: `.env.local`이라는 금고에 `GEMINI_API_KEY`라는 열쇠를 새로 만들어서 넣어주었습니다. 이제 AI라는 집사가 문을 열고 일할 수 있습니다.

### 주의사항
- API 키는 남들에게 절대 보여주면 안 되는 비밀번호입니다. 그래서 보통 코드 파일에 직접 적지 않고 `.env` 같은 별도의 설정 파일에 숨겨둡니다.

## 4. 데이터 저장: 단기 기억 vs 장기 기억 (Local Storage)

### 상황
앱을 새로고침하면 열심히 쓴 아이디어와 계획이 다 사라지는 문제가 있었습니다.

### 비유: "칠판 vs 공책"
- **이전 (Memory State)**: 칠판에 적어두는 것과 같습니다. 수업(앱 실행)이 끝나고 청소(새로고침)하면 싹 지워집니다.
- **해결 (Local Storage)**: 브라우저 안에 **개인 사물함(Local Storage)**을 만들었습니다.
    - 앱이 켜질 때 사물함을 먼저 열어서 데이터가 있는지 봅니다.
    - 무언가 적으면 즉시 사물함에 넣어둡니다.
    - 이제 창을 닫았다가 다시 열어도 사물함에서 데이터를 꺼내오므로 기억이 생생합니다!

### 꿀팁: 내 사물함 훔쳐보기 (Where is it?)
로컬 스토리지는 내 컴퓨터 폴더(Finder)에 있는 파일이 아니라, **"크롬 브라우저 안의 비밀 주머니"**입니다.
눈으로 직접 확인해 볼 수 있어요:
1. 앱 화면에서 우클릭 -> **"검사 (Inspect)"** 클릭
2. 나타나는 창 상단 메뉴에서 **"Application"** 탭 클릭
   (안 보이면 `>>` 화살표를 눌러보세요)
3. 왼쪽 사이드바에서 **"Local Storage"** -> `http://localhost:3000` 클릭
4. 여기에 저장된 `Key`와 `Value`들이 보입니다!

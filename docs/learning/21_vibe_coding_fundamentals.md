# 비개발자를 위한 바이브코딩(Vibe Coding) 펀더멘탈

## 바이브코딩이란 무엇인가?
바이브코딩은 단순히 기능(Function)을 구현하는 것이 아닙니다. **감정(Emotion)과 경험(Experience)**을 구현하는 것입니다.
우리가 오늘 'AI 코치'를 만들 때, "채팅 기능이 필요해"가 아니라 "일론 머스크한테 깨지는 느낌을 받고 싶어"라는 **Vibe**에서 시작했습니다.

이것이 바이브코딩의 핵심입니다.

## 1. 페르소나(Persona)가 기능을 정의한다
- **일반적인 코딩**: "AI 챗봇 서비스 생성" -> "텍스트 입력" -> "텍스트 출력"
- **바이브코딩**: "일론 머스크라면 어떻게 말할까?" -> "First Principles" -> "디자인은 차갑고 미래지향적인 Cyan 컬러"

우리는 코드를 짜기 전에 **'누구(Who)'**와 **'어떤 느낌(Feel)'**인지를 먼저 정의했습니다.
- **Elon**: 물리 법칙, 효율성, Cyan/Black, Glitch 효과
- **Goggins**: 고통, 규율, Red/Military, 강렬한 폰트

기능은 이 느낌을 전달하기 위한 수단일 뿐입니다.

## 2. 디테일이 설득력을 만든다 (The Devil is in the Detail)
사용자가 "이건 진짜다"라고 느끼게 하는 것은 거창한 AI 알고리즘보다 사소한 UI 디테일인 경우가 많습니다.
- **Micro-Copy**: 로딩 중일 때 단순히 "Loading..."이 아니라 "CALCULATING..."(Elon)이나 "TRAINING..."(Goggins)이라고 뜨는 것.
- **Color Psychology**: 일론은 냉철한 블루/사이언, 고긴스는 피와 땀의 레드.
- **Fallback Experience**: API 에러가 났을 때조차 "에러가 발생했습니다"가 아니라 "API 오류? 좋아, 다시 시작해! STAY HARD!"라고 외치는 것.

## 3. 완벽하지 않아도, 매력적이어야 한다
오늘 우리가 만든 코치는 실제 지능을 가진 AI가 아닐 수도 있습니다(API 연결이 없더라도). 하지만 사용자는 그 **'컨셉'**에 몰입합니다.
비개발자가 AI와 협업할 때 가장 강력한 무기는 **"어떤 경험을 주고 싶은가?"**를 구체적으로 상상하는 능력입니다.

> "기능은 복제할 수 있지만, 바이브는 복제할 수 없다."

이 노트가 당신의 다음 아이디어를 실현하는 데 도움이 되길 바랍니다.
Stay Hard, Think First Principles.

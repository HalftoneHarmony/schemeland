# 19. 대청소 후 사라진 방 찾기 (View 연결 복구)

> 🍼 **응애 개발자를 위한 요약**: 
> 집(`App.tsx`) 리모델링을 하다가 실수로 **방문**을 안 만들어서, 방에 못 들어가는 사태가 발생했습니다.
> 다시 **문을 달아주고**, 기왕 들어간 김에 방 안의 **오래된 배선**도 새걸로 교체했습니다!

---

## 1. 무슨 일이 있었나요? 😱

우리가 지난번에 `App.tsx`라는 거대한 파일을 정리했잖아요? (이걸 **리팩토링**이라고 해요.)
너무 복잡한 코드를 싹 정리해서 깔끔하게 만들었는데, 그 과정에서 실수로 **두 개의 화면(View)**을 연결하는 코드를 빼먹었어요.

*   **사라진 방 1**: `Campaign Detail` (미션 상세 화면)
*   **사라진 방 2**: `Kanban` (태스크 매트릭스 화면)

그래서 버튼을 눌러도 "어? 왜 아무 반응이 없지?" 했던 거예요. 문이 벽으로 막혀 있었거든요.

## 2. 어떻게 고쳤나요? 🛠️

### 1) 방문 다시 달기 (Routing 복구)
`App.tsx`는 우리 앱의 **안내 데스크** 같은 곳이에요.
*"지금 상태가 `KANBAN`이면, 칸반 화면을 보여줘!"* 라고 말하는 코드를 다시 넣어줬어요.

```tsx
// App.tsx 안에 이런 안내판을 다시 세웠어요
if (currentView === 'KANBAN') {
    return <KanbanView ... />;
}
```

### 2) 방 안 구조 변경 (배선 공사)
`KanbanView` 방에 들어가 봤더니, 옛날 방식(Legacy)으로 데이터를 고치고 있었어요.
*   **옛날 방식**: "책상 하나 옮기려면, 집 전체를 들어서 옮긴다." (비효율적)
*   **새 방식**: "책상만 슥 옮긴다." (Store 액션 사용)

그래서 기왕 고치는 김에, 칸반 보드가 **더 가볍고 빠르게** 동작하도록 내부 배선을 최신식(`Store` 직접 연결)으로 바꿨습니다.

## 3. 바이브코딩 꿀팁 🍯

### "리팩토링(Refactoring)은 양날의 검"
*   코드를 정리하면 보기는 좋아지지만, 이렇게 **실수로 기능을 빼먹는 사고**가 종종 발생해요.
*   그래서 개발자들은 **"수술(코드 수정) 끝나고 환자(앱)가 숨을 잘 쉬는지(모든 화면이 나오는지) 꼭 확인"**해야 합니다.

### "Props 드릴링 줄이기"
*   칸반 보드 고칠 때, `App.tsx`에서 모든 도구를 다 챙겨서 보내주는 대신, 칸반 보드가 직접 공구함(`Store`)을 열어서 쓰게 만들었어요. 
*   이렇게 하면 `App.tsx`가 훨씬 가벼워집니다!

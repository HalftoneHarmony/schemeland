# 4. 데이터 저장: 단기 기억 vs 장기 기억 (Local Storage)

### 상황
앱을 새로고침하면 열심히 쓴 아이디어와 계획이 다 사라지는 문제가 있었습니다.

### 비유: "칠판 vs 공책"
- **이전 (Memory State)**: 칠판에 적어두는 것과 같습니다. 수업(앱 실행)이 끝나고 청소(새로고침)하면 싹 지워집니다.
- **해결 (Local Storage)**: 브라우저 안에 **개인 사물함(Local Storage)**을 만들었습니다.
    - 앱이 켜질 때 사물함을 먼저 열어서 데이터가 있는지 봅니다.
    - 무언가 적으면 즉시 사물함에 넣어둡니다.
    - 이제 창을 닫았다가 다시 열어도 사물함에서 데이터를 꺼내오므로 기억이 생생합니다!

### 꿀팁: 내 사물함 훔쳐보기 (Where is it?)
로컬 스토리지는 내 컴퓨터 폴더(Finder)에 있는 파일이 아니라, **"크롬 브라우저 안의 비밀 주머니"**입니다.
눈으로 직접 확인해 볼 수 있어요:
1. 앱 화면에서 우클릭 -> **"검사 (Inspect)"** 클릭
2. 나타나는 창 상단 메뉴에서 **"Application"** 탭 클릭
   (안 보이면 `>>` 화살표를 눌러보세요)
3. 왼쪽 사이드바에서 **"Local Storage"** -> `http://localhost:3000` 클릭
4. 여기에 저장된 `Key`와 `Value`들이 보입니다!

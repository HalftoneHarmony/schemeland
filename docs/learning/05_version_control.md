# 5. 버전 컨트롤 (Version Control - Git & GitHub)

### Q: Antigravity가 버전 관리를 알아서 해주나요?
**A: 반은 맞고 반은 틀립니다.**
Antigravity는 전 세계 표준인 **Git(깃)**을 사용합니다. 제가 내부적으로 독자적인 시스템을 쓰는 게 아니라, 개발자들이 쓰는 **진짜 Git 명령어**를 대신 입력해 드리는 것입니다.

지금 제가 프로젝트 폴더에서 `git init`, `git commit`을 해서 로컬(내 컴퓨터)에 저장소는 만들었습니다. 하지만 **GitHub(인터넷 저장소)**에 올리는 건 사용자의 계정 권한이 필요해서, 최초 1회 연결 작업이 필요합니다.

### 🚀 GitHub에 내 프로젝트 올리는 법 (따라해 보세요!)

1.  **GitHub 저장소 만들기**
    *   [GitHub.com](https://github.com)에 로그인합니다.
    *   우측 상단 `+` 버튼 -> `New repository` 클릭.
    *   **Repository name**: `schemeland` (또는 원하는 이름)
    *   Public/Private 선택 후 `Create repository` 클릭.

2.  **연결 주소 복사하기**
    *   생성된 화면에서 `HTTPS` 주소를 복사합니다. 
    *   예: `https://github.com/사용자ID/schemeland.git`

3.  **Antigravity에게 주소 알려주기**
    *   채팅창에 이렇게 말해주세요:
    *   `"깃허브 주소는 https://github.com/사용자ID/schemeland.git 야. 여기로 푸시해줘."`

### 💡 용어 정리
*   **Git (깃)**: 내 컴퓨터에서 파일의 역사를 기록하는 타임머신.
*   **GitHub (깃허브)**: 그 기록을 안전하게 보관하는 구글 드라이브 같은 클라우드.
*   **Commit (커밋)**: "이 시점으로 저장해!" 하고 사진 찍는 것 (세이브 포인트).
*   **Push (푸시)**: 내 컴퓨터의 세이브 파일을 깃허브로 업로드하는 것.

### 📅 언제 Git을 써야 하나요? (Good Habits)

**Q: "얼마나 자주 저장(Commit)해야 돼?"**

#### 1. "의미 있는 단위"로 완성했을 때 (Save Game)
게임을 하다가 보스를 깼거나, 중요한 아이템을 얻으면 바로 세이브를 하죠? 코딩도 똑같습니다.
*   ✅ "로그인 기능 완성했어!" -> **Commit**
*   ✅ "버튼 디자인 예쁘게 바꿨어!" -> **Commit**
*   ✅ "오타 수정했어." -> **Commit**

#### 2. "불안할 때" (Checkpoint)
"이거 건드리면 다 망가질 것 같은데...?" 하는 어려운 작업을 시작하기 전이나, 작업 도중 잘 돌아가는 상태일 때 일단 저장하세요. 망하면 바로 돌아올 수 있습니다.

#### 3. 하루를 마무리는 때 (Sync)
작업을 끝내고 컴퓨터를 끄기 전에는 꼭 **Commit & Push**를 하는 습관을 들이세요.
*   내 컴퓨터가 갑자기 고장 나도 깃허브에 코드가 살아있습니다.
*   다음 날 다른 컴퓨터에서도 이어서 작업할 수 있습니다.

### ✨ 이것만은 지켜주세요 (Best Practices)
1.  **커밋 메시지는 솔직하고 명확하게**: "점심 먹고 수정" (X) -> "메인 화면 로고 크기 변경" (O)
2.  **에러가 있는 채로 커밋하지 않기**: 세이브 파일을 로드했는데 게임이 켜지자마자 꺼지면 안 되겠죠? 실행은 되는 상태에서 저장하는 게 좋습니다.

# 📊 데이터 엔지니어링: 프로젝트의 뼈대와 질서 잡기

비개발자도 이해할 수 있는 **데이터 구조화(Data Structuring)** 및 **프로젝트 정리** 가이드입니다.

---

## 1. 📂 집 안 정리: `src/` 폴더로 모으기
**"부엌 여기저기 흩어져 있던 도구들을 한 수납장(`src`)에 모았습니다."**

- **작업 내용**: 루트 경로에 있던 `App.tsx`, `components`, `services` 등을 모두 `src/` 폴더 안으로 옮겼습니다.
- **이유**: 프로젝트가 커지면 파일이 수백 개가 될 수 있습니다. 이때 소스 코드는 따로 모아두어야 관리하기 편하고, 다른 개발(혹은 AI)이 파일을 찾을 때 헷갈리지 않습니다. 이것은 현대 웹 개발의 표준(Standard)입니다.

## 2. 뼈대 세우기: `BaseEntity`와 엄격한 타입
**"모든 서류에 반드시 이름, 작성일, 수정일 도장을 찍기로 했습니다."**

- **작업 내용**: `BaseEntity`라는 기본 공통 양식을 만들고, 모든 데이터(아이디어, 프로젝트 등)가 이를 따르도록 했습니다.
- **이유**: 
    - 나중에 서버(Database)에 정보를 저장할 때 "이 데이터는 언제 만들어졌지?"라는 질문에 답하려면 모든 데이터에 생성일(`createdAt`)이 있어야 합니다.
    - **엄격한 타입**: 단순히 '글자'라고 하지 않고, '진행중', '완료' 처럼 정해진 값만 넣을 수 있게 했습니다. 초보자가 실수로 '할일' 대신 '하는중'이라고 오타를 내면 코드가 즉시 알려주어 오류를 방지합니다.

## 3. 💾 자동 저장 비서: `useLocalStorage` 커스텀 훅
**"파일을 캐비닛에 넣고 뺄 때마다 내용이 맞는지 검사하는 비서를 두었습니다."**

- **작업 내용**: `useLocalStorage.ts`라는 도구를 만들어, 브라우저 저장소에 데이터를 저장할 때 자동으로 우리가 정한 규칙(타입)을 지키는지 확인하게 했습니다.
- **이유**: 
    - 매번 복잡하게 코드를 짜지 않아도 됩니다. 이 도구 하나면 안전하게 데이터를 저장할 수 있습니다.
    - 앱을 껐다 켜도 데이터가 유지되면서도, 데이터 형식이 깨지지 않도록 보호합니다.

---

## 💡 학습 포인트: "데이터 엔지니어가 왜 필요한가?"
개발을 하다 보면 눈에 보이는 화면(UI/UX)에 집중하기 쉽습니다. 하지만 **데이터 엔지니어**는 보이지 않는 지하의 **상하수도와 배선**을 깔고 있는 사람입니다.

- **지금 이 작업을 하지 않으면?**: 나중에 서버를 연결할 때 모든 코드를 다 뜯어고쳐야 하거나, 데이터 형식이 엉망이라 앱이 갑자기 꺼지는 일이 빈번해집니다.
- **결과**: 오늘 한 작업 덕분에 이제 이 프로젝트는 실제 서비스를 제공하는 **전문적인 앱**으로 성장할 준비가 끝났습니다.
